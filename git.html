git-branch
git branch [--list] [-r | -a]list existing branches
-r list remote-tracking branches
-a list local and remote branches
git branch new-branchname [start-point] create new branch new-branchname which points to the current HEAD, or start-point if given (branch name, commit-id, or tag)
git branch -d branchname [-r] [-f]delete a branch (the branch must be fully merged)
-r delete the remote-tracking branch
-f --force allow deleting the branch irrespective of its merged status
git branch -D branchname [-r]delete a branch irrespective of its merged status (similar to -d --force)
-r delete the remote-tracking branch
git branch -m old-branchname new-branchname [-f]move/rename a branch
-f --force allow renaming the branch even if the new branch name already exists
git branch -M old-branchname new-branchname move/rename a branch even if the new branch name already exists(similar to -m --force)
git branch -c old-branchname new-branchname [-f]copy a branch
-f --force allow coping the branch even if the new branch name already exists
git branch -C old-branchname new-branchname copy a branch even if the new branch name already exists(similar to -c --force)
git-checkout
git checkout branchname [-f]switch to branch
-f --force proceed even if the index or the working tree differs from HEAD (throw away local changes)
git checkout -b new-branchname [start-point]switch to new created branch which starts at HEAD, or start-point if given (branch name, commit-id, or tag)
git checkout -B new-branchname [start-point]switch to new created branch which starts at HEAD, or start-point if given (branch name, commit-id, or tag); if it already exists, then reset it to start-point
git-tag
git tag -a tagname--annotate make an unsigned, annotated tag object
git tag -d tagname--delete delete existing tag with the given name
git tag -l--list list tags
git-merge
git merge branchname [--ff | --no-ff] incorporate changes from the branchname into the current branch
(replay the changes made on the branchname since it diverged from current branch until its last commit on top of current branch, and record the result in a new commit)
--ff (fast-forward) only update the branch pointer, without creating a merge commit (default behavior)
--no-ff create a merge commit even when the merge resolves as a fast-forward
git merge --continueconclude the merge after a git merge stops due to conflicts
git merge --abortabort the current conflict resolution process, and try to reconstruct the pre-merge state
git-push
git push remote [branchname] [-f] push the specified branch (or current branch if not specified) from local repository to remote repository;
this creates a local branch in the remote repository;
command refuses to update a remote repository that is not an ancestor of the local repository
-f --force disable the ancestor check (can cause the remote repository to lose commits)
git push remote --allpush all local branches to the remote repository
git push remote --tagspush all local tags to the remote repository
Gitflow
git flow init
git flow feature start feature-branch
git flow feature finish feature-branch
git flow release start x.x.x
git flow release finish x.x.x
git flow hotfix start hotfix-branch
git flow hotfix finish hotfix-branch
develop branch is created from master
feature branch is created from develop
release branch is created from develop
hotfix branch is created from master

feature branch is merged into develop
release branch is merged into develop and master
hotfix branch is merged into develop and master
